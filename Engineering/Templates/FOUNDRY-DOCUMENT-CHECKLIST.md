# Purpose

This checklist defines the minimum documentation required for every Cougar Foundry.

Every Foundry should follow a consistent structure so that it can be understood, implemented, maintained, and expanded over time.

This checklist applies to both current and future Foundries across the Cougar Platform.

# Completion Criteria

A Foundry is considered complete when:

- Core documents are finished.
- Architecture is documented.
- Governance is defined.
- Engineering standards are met.
- Security has been reviewed.
- Documentation has been approved.

---

# Guiding Principle

> Every Foundry should be understandable by a new engineer years from now and should provide enough documentation to support continued evolution without relying on tribal knowledge.
